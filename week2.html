Week 2 – Security Planning and Testing Methodology

Performance Testing Plan

Remote monitoring will be conducted via SSH using command-line tools such as top, htop, vmstat, and iostat to measure CPU usage, memory usage, and disk activity. Baseline measurements will be taken before workload testing to provide a comparison point.

Security Configuration Checklist

  • SSH hardening: key-based authentication, disable root login, restrict access where possible.
  • Firewall configuration: allow SSH only from the workstation IP, block all other inbound traffic.
  • User privilege management: create a non-root admin user with sudo, least privilege.
  • Automatic updates: enable security updates automatically.
  • Network security: limit exposed services/ports and monitor access attempts.

Threat Model

Key threats identified include SSH brute-force attempts, unauthorised access, and service exploitation via open ports or misconfiguration. Mitigations include key-based SSH authentication, firewall restrictions, and fail2ban to detect and block repeated login attempts.

Reflection

Planning security controls early ensures the system is deployed with a clear baseline. This reduces risk during later phases when services are tested under load and makes audit evidence easier to collect.
