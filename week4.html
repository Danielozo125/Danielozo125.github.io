Week 4 – Security Hardening

Objectives

  • Configure SSH using key-based authentication
  • Disable password login to reduce brute-force risk
  • Apply and document security configuration changes

SSH Key-Based Authentication

SSH was configured to use key-based authentication by ensuring a valid public key exists in ~/.ssh/authorized_keys for the administrative user. This approach improves security by eliminating weak password-based authentication.

Password Login Disabled

Password-based SSH authentication was disabled in /etc/ssh/sshd_config by setting: PasswordAuthentication no. This prevents brute-force password attacks and enforces the use of SSH keys.

Figure 1: SSH hardening evidence showing PasswordAuthentication no in /etc/ssh/sshd_config.

Reflection

This week improved the baseline security of the server by enforcing stronger authentication controls. Disabling password authentication reduces the attack surface and supports secure remote administration for the remaining coursework phases.
