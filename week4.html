Week 4 – Security Hardening

Objectives

  • Configure secure SSH access using key-based authentication
  • Disable password-based login
  • Restrict access using firewall rules

SSH Key-Based Authentication

SSH access was secured by configuring public key authentication and disabling password-based login. This ensures only authorised users with a valid private key can access the system.

Figure 1: SSH configuration showing password authentication disabled.

Firewall Configuration (UFW)

The Uncomplicated Firewall (UFW) was configured to restrict SSH access to a specific trusted IP address. This significantly reduces the attack surface and prevents unauthorised remote access.

Figure 2: UFW rule allowing SSH access only from a trusted IP address.

Summary

These security measures strengthen the system by enforcing strong authentication and limiting network exposure. The configuration ensures that only authorised users can access the server while reducing the risk of attack.
