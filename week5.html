Week 5 – Security Hardening (Fail2Ban)

Objective

The objective of this phase was to implement intrusion prevention mechanisms to protect the server from brute-force attacks, particularly targeting SSH.

Fail2Ban Configuration

Fail2Ban was installed and configured to monitor SSH authentication logs. When repeated failed login attempts were detected, the offending IP address was automatically blocked.

Verification

The screenshot above confirms that Fail2Ban is active and has successfully banned an IP address after multiple failed login attempts. This significantly reduces the risk of brute-force attacks on the server.

Conclusion

Implementing Fail2Ban enhances system security by actively responding to suspicious activity and enforcing automated protection mechanisms.
