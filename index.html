Project Overview

This journal documents the design, deployment, security hardening, monitoring, performance testing, and evaluation of a Linux-based server environment. The project follows a structured 7-week assessment plan and demonstrates progressive learning through practical implementation and analysis.

Week 1 – System Planning & Distribution Selection

  • System architecture design using VirtualBox
  • Server and workstation role separation
  • Linux distribution comparison and justification
  • Network configuration and IP addressing
  • System specification collection using CLI commands

Week 2 – Security Planning & Threat Modelling

  • Security baseline definition
  • SSH hardening strategy
  • Firewall and access control planning
  • Threat model identifying key risks and mitigations

Week 3 – Application Selection

  • CPU-intensive workload selection
  • Memory-intensive workload selection
  • Disk I/O testing applications
  • Network and server-based applications
  • Expected resource usage profiling

Week 4 – Initial System Configuration & Security

  • SSH key-based authentication
  • Firewall configuration with restricted access
  • User privilege management
  • Remote administration via SSH

Week 5 – Advanced Security & Monitoring

  • SELinux / AppArmor access control
  • Automatic security updates
  • Fail2Ban intrusion prevention
  • Custom security baseline verification script
  • Remote monitoring script implementation

Week 6 – Performance Evaluation

  • Baseline performance testing
  • Load testing and monitoring
  • Bottleneck identification
  • System optimisation and improvements
  • Performance data tables and graphs

Week 7 – Security Audit & System Evaluation

  • Lynis security audit
  • Nmap network scanning
  • Service inventory and justification
  • Final risk assessment
  • Overall system evaluation
